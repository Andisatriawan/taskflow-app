TaskFlow — API Design Reference

Phase 3 — System Architecture | All REST API endpoints | Version 1.0 | 22 March 2026
GET
Fetch data
POST
Create new
PATCH
Update existing
DELETE
Remove
admin Admin only
Authentication
POST
/api/auth/login
Login with email & password — returns JWT token
POST
/api/auth/logout
Logout — invalidates current session token
GET
/api/auth/me
Get current logged-in user profile and role
Users
GET
/api/users
Get all staff members admin
GET
/api/users/:id
Get one user's profile details admin
POST
/api/users
Create a new staff account admin
PATCH
/api/users/:id
Update staff name, email or active status admin
Tasks
GET
/api/tasks
Get tasks — admin sees all, staff sees own tasks only
GET
/api/tasks/:id
Get full detail of one task including notes and files
POST
/api/tasks
Create and assign a new task to a staff member admin
PATCH
/api/tasks/:id
Edit task title, description, due date or priority admin
PATCH
/api/tasks/:id/status
Update task status — staff updates their own tasks only
PATCH
/api/tasks/:id/verify
Verify or reject a completed task admin
DELETE
/api/tasks/:id
Permanently delete a task admin
Task Notes
GET
/api/tasks/:id/notes
Get all notes for a specific task
POST
/api/tasks/:id/notes
Add a new note or comment to a task
Task Files
GET
/api/tasks/:id/files
Get all file attachments for a task
POST
/api/tasks/:id/files
Upload a file attachment to a task (stored in Cloudinary)
DELETE
/api/tasks/:id/files/:fileId
Delete a specific file attachment admin
