1
Sarah opens the dashboard
Layer: React frontend
Sarah loads taskflow.app in her browser. React renders the dashboard page. The component needs task data — so it fires a request using Axios. This happens automatically when the page loads.
// In React — TaskDashboard.jsx useEffect(() => { fetchTasks(); // runs when page loads }, []); const fetchTasks = async () => { const response = await axios.get('/api/tasks'); setTasks(response.data); // store in React state };
2
Axios sends the request
Layer: Axios (the messenger)
Axios takes the request and sends it over the internet to the backend server using HTTPS. It also automatically attaches Sarah's login token (JWT) to the request header — so the backend knows who is asking.
// Axios config — set up once in api.js const api = axios.create({ baseURL: 'https://taskflow-api.railway.app', headers: { 'Authorization': 'Bearer ' + userToken // Sarah's JWT login token — proves who she is } }); // Every request automatically includes this token
3
Backend receives the request
Layer: Express backend (Node.js)
The Express server receives the request at the /api/tasks route. First it checks the JWT token — is this a real logged-in user? Is she an admin? If yes, it proceeds. If not, it immediately returns a 401 Unauthorized error.
// In Express — tasks.routes.js router.get('/api/tasks', verifyToken, // check JWT first checkAdmin, // is this Sarah (admin)? async (req, res) => { // only reaches here if token is valid const tasks = await prisma.task.findMany(); res.json(tasks); // send tasks back } );
Security rule: Even if someone hacks the React code, the backend always checks the real role from the database. All security lives in the backend — never in the frontend.
4
Prisma queries the database
Layer: Prisma + PostgreSQL
Express tells Prisma to fetch all tasks from the PostgreSQL database. Prisma translates this into a SQL query, runs it, and returns the results as clean JavaScript objects — no complex SQL needed!
// Prisma query — reads like plain English const tasks = await prisma.task.findMany({ include: { assignedTo: true, // include staff name createdBy: true, // include who made it }, orderBy: { due_date: 'asc' // sorted by due date } }); // returns array of task objects
5
Response travels back to React
Layer: Backend → React
The backend sends the tasks back as JSON data over HTTPS. Axios receives it on the frontend. React stores it in state and automatically re-renders the dashboard — Sarah now sees her task list on screen.
// Backend sends this JSON response: [ { id: "abc-123", title: "Write client report Q1", assignedTo: { name: "Budi" }, status: "IN_PROGRESS", due_date: "2026-03-24", priority: "HIGH" }, // ... more tasks ] // React receives it → dashboard updates instantly
6
The full picture — complete flow
Summary of the entire request cycle
Every button click and page load in the app follows this exact same pattern. The only thing that changes is the endpoint URL and what data is sent or received.
SARAH'S BROWSER INTERNET SERVER + DATABASE ───────────────── ──────── ────────────────────── React component │ └─ axios.get() ──── HTTPS request ──→ Express router (with JWT) │ verifyToken() │ Prisma query │ setTasks(data) <─── JSON response ─── res.json(tasks) │ └─ render() → Sarah sees tasks on screen
The golden rule: React never talks directly to the database. It always goes through the API. This keeps data safe and the app organized.
